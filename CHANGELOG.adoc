= Changelog
:toc: left

This document details all notable changes and releases for `asciidocstring`.

== [0.1.0a5] - 2026-07-17

=== Fixed
* **Consecutive Definition List Items**: Ensured standard reST formatting by writing exactly one separating blank line between consecutive items in description lists.
* **Uniform Multi-line Indentation**: Aligned multi-line paragraphs uniformly under nested block structures (e.g. description list definitions), preventing Docutils blockquote parsing errors.

=== Metrics
* Reached **100% total code coverage** across all library modules.

---

== [0.1.0a4] - 2026-07-16

=== Added
* **Safe Parsing Mode**: Introduced a `safe_mode` parameter to `parse()` and `AsciiDocStringDocument`. When `safe_mode=True`, syntax errors and general parsing failures emit a non-blocking `AsciiDocStringWarning` and return a rich structured fallback AST.
* **Inline Caret Fallback**: The fallback warning AST compiles to a standard Sphinx-compatible `.. warning::` admonition block enclosing a pre-formatted `.. code-block:: asciidoc` block with a dynamically inserted caret cursor pointing to the exact column of the syntax error.
* **API Warning Class**: Exposed `AsciiDocStringWarning` at the package-level.

=== Fixed
* **TestBlock Documentation Mismatch**: Aligned attributes of the `TestBlock` class documented in `README.adoc` with the real implementation (`content`, `language`, `line_number`, `is_interactive`, `attributes`).
* **Quick Start Sample**: Corrected reference from non-existent `block.code` to `block.content` in the documentation's quick start example.

---

== [0.1.0a3] - 2026-07-14

=== Added
* **Structured Parse Errors**: Exposed our own new structured exception `AsciiDocStringParseError` that wraps `asciidoctrine`'s new `AsciiDocSyntaxError` to detail line and column numbers alongside highlighted caret context.
* **Unified Preprocessor & Parse Exception Handling**: Centralized all exceptions within the `AsciiDocStringDocument.__init__` constructor, ensuring that any pre-processing, formatting, or parsing failures are caught and wrapped cleanly.

=== Upgraded
* Bumped minimum `asciidoctrine` dependency constraint to `v0.1.0a8` in `pyproject.toml`.

=== Metrics
* Maintained **99% total code coverage** (with 100% coverage on the preprocessor and document parsing module).

---

== [0.1.0a2] - 2026-07-13

=== Added
* **Native Footnotes Support**: Integrated complete tracking, referencing, and serialization of anonymous footnotes (`footnote:[...]`) and named footnote references/definitions (`footnoteref:[id, text]`, `footnoteref:[id]`), appending standard reST footnote definitions at the bottom of the rendered document.
* **Testing Optional Dependencies**: Declared `pytest` and `pytest-cov` under `[project.optional-dependencies]` in `pyproject.toml` so developers can run `pip install -e .[test]` to get the full testing stack.
* **Coverage Tracking**: Integrated `pytest-cov` and established a baseline test coverage metric.

=== Fixed
* **Thematic Break Visitor Dispatch**: Renamed `visit_thematicbreak` to `visit_thematic_break` in `ReSTSerializerVisitor` to align with the lowercased AST name `thematic_break` dispatched by `NodeVisitor`, correcting a silent parsing/rendering bug.
* **Package Entry Documentation**: Resolved documentation gaps by adding package-level docstrings to `src/asciidocstring/__init__.py`.

=== Upgraded
* Bumped minimum `asciidoctrine` dependency constraint to `v0.1.0a7` in `pyproject.toml`.

=== Metrics
* Reached **99% total code coverage** across all modules.

---

== [0.1.0a1] - 2026-07-11

=== Added
* Initial release of `asciidocstring`, a pure-Python, Pyodide-compatible AsciiDoc docstring parsing and rendering utility.
* **Docstring Preprocessor**: Strips common leading indentation using `inspect.cleandoc` and normalizes trailing lines to prevent Lark EOF errors.
* **ReST Serialization Visitor**: Recursively translates AsciiDoc docstrings into high-fidelity reStructuredText (reST) compatible with Sphinx. Supports headers, nested lists, definition lists, styled text spans, admonitions, and inline links.
* **Test Block Extractor Visitor**: Extracts executable code blocks from the docstring ASG. Supports interactive `>>>` prompts and handles both inline/positional test markers.
